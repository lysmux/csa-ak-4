# Ввод / Вывод

Модель ввода-вывода — **memory-mapped + trap**:

* Гарвардская архитектура: MMIO размещается в адресном пространстве **памяти данных**, память команд этим не затрагивается
* Отдельных команд `IN`/`OUT` нет — устройства видны как обычные ячейки и читаются/пишутся командами `LOAD addr`, `STORE addr`
* Асинхронные события (готовность токена, освобождение буфера, истечение таймера) сигнализируются через систему прерываний (trap).

# Карта памяти данных

Адрес — 24 бита (операнд команды занимает 3 байта). Верхние 256 байт зарезервированы под MMIO, всё остальное — пользовательская память данных.

```
0x000000 ┐
         │ Таблица векторов прерываний (5 × 4 = 20 байт, 0x000000..0x000013)
0x000013 ┘
0x000014 ┐
         │
         │ Память данных пользователя
         │
0xFFFEFF ┘
0xFFFF00 ┐
         │ MMIO (256 байт)
0xFFFFFF ┘
```

# MMIO-регион

Блоки выровнены по 16 байт; каждое устройство занимает свой блок:

| Диапазон              | Устройство                |
|-----------------------|---------------------------|
| `0xFFFF00..0xFFFF03`  | Входной порт `IN0`        |
| `0xFFFF04..0xFFFF07`  | Входной порт `IN1`        |
| `0xFFFF10..0xFFFF13`  | Выходной порт `OUT0`      |
| `0xFFFF14..0xFFFF17`  | Выходной порт `OUT1`      |
| `0xFFFF20..0xFFFF23`  | Таймер                    |
| `0xFFFF30..0xFFFF33`  | Контроллер прерываний     |

Адреса остальных смещений зарезервированы; чтение даёт 0, запись игнорируется.

## Входные порты (IN0, IN1)

Каждый порт — 4-байтовый блок. Поток ввода токен-ориентированный: 1 ячейка = 1 байт-токен. Для `cstr`-строк токены читаются по одному до `0x00`.

| Смещение | Регистр      | Доступ | Назначение                                                          |
|----------|--------------|--------|---------------------------------------------------------------------|
| +0       | `INx_DATA`   | R      | Следующий токен из FIFO. Чтение снимает токен. Запись игнорируется. |
| +1       | `INx_STATUS` | R      | Биты состояния (см. ниже).                                          |
| +2       | `INx_CTRL`   | R/W    | Управляющие биты (см. ниже).                                        |
| +3       | —            | —      | Резерв.                                                             |

`INx_STATUS`:

| Бит | Имя   | Смысл                                                  |
|-----|-------|--------------------------------------------------------|
| 0   | `RDY` | В FIFO есть хотя бы один непрочитанный токен.          |
| 1   | `OVF` | Произошло переполнение буфера (sticky).                |
| 2   | `EOF` | Поток закрыт (после опустошения FIFO новых не будет).  |

`INx_CTRL`:

| Бит | Имя       | Смысл                                                    |
|-----|-----------|----------------------------------------------------------|
| 0   | `IRQ_EN`  | Разрешение прерывания «данные доступны» от этого порта.  |
| 1   | `CLR_OVF` | Запись 1 сбрасывает `OVF` (read-as-0).                   |

Чтение `INx_DATA` при `RDY = 0` возвращает 0 и не считается ошибкой; программе следует сначала проверять `RDY` (либо работать через прерывание).

## Выходные порты (OUT0, OUT1)

| Смещение | Регистр       | Доступ | Назначение                                                |
|----------|---------------|--------|-----------------------------------------------------------|
| +0       | `OUTx_DATA`   | W      | Запись байта в очередь устройства.                        |
| +1       | `OUTx_STATUS` | R      | Биты состояния (см. ниже).                                |
| +2       | `OUTx_CTRL`   | R/W    | Управляющие биты (см. ниже).                              |
| +3       | —             | —      | Резерв.                                                   |

`OUTx_STATUS`:

| Бит | Имя    | Смысл                                              |
|-----|--------|----------------------------------------------------|
| 0   | `RDY`  | Буфер готов принять следующий байт.                |
| 1   | `BUSY` | Передача в процессе.                               |

`OUTx_CTRL`:

| Бит | Имя      | Смысл                                                       |
|-----|----------|-------------------------------------------------------------|
| 0   | `IRQ_EN` | Прерывание по переходу `RDY` 0 → 1 (буфер освободился).     |

Запись в `OUTx_DATA` при `RDY = 0` ставит байт в очередь, если в устройстве есть место; в противном случае поведение зависит от устройства (тестовый стенд эмулятора расширяет очередь).

## Таймер

| Смещение | Регистр      | Доступ | Назначение                                                |
|----------|--------------|--------|-----------------------------------------------------------|
| +0       | `TMR_VAL`    | R/W    | Текущее значение счётчика. Декрементируется на 1 каждый такт, если `ENABLE = 1`. |
| +1       | `TMR_RELOAD` | R/W    | Значение, в которое грузится счётчик при достижении 0, если `AUTO = 1`. |
| +2       | `TMR_CTRL`   | R/W    | Управляющие биты (см. ниже).                              |
| +3       | —            | —      | Резерв.                                                   |

`TMR_CTRL`:

| Бит | Имя      | Смысл                                                            |
|-----|----------|------------------------------------------------------------------|
| 0   | `ENABLE` | Разрешение работы счётчика.                                      |
| 1   | `AUTO`   | Автоперезагрузка из `TMR_RELOAD` при обнулении.                  |
| 2   | `IRQ_EN` | Запрашивать прерывание при обнулении счётчика.                   |
| 3   | `PEND`   | Sticky-флаг истечения; запись 1 сбрасывает.                      |

## Контроллер прерываний

| Смещение | Регистр       | Доступ | Назначение                                                          |
|----------|---------------|--------|---------------------------------------------------------------------|
| +0       | `IRQ_PENDING` | R      | Битовая маска источников с активным запросом.                       |
| +1       | `IRQ_MASK`    | R/W    | Биты разрешения по источникам (1 — источник участвует в арбитраже). |
| +2       | `IRQ_ACK`     | W      | Запись 1 в бит сбрасывает соответствующий бит в `IRQ_PENDING`.      |
| +3       | —             | —      | Резерв.                                                             |

Нумерация источников (один и тот же номер бита во всех трёх регистрах):

| Бит | Источник | Условие запроса                                          |
|-----|----------|----------------------------------------------------------|
| 0   | `IN0`    | `IN0_STATUS.RDY = 1` ∧ `IN0_CTRL.IRQ_EN = 1`             |
| 1   | `IN1`    | `IN1_STATUS.RDY = 1` ∧ `IN1_CTRL.IRQ_EN = 1`             |
| 2   | `OUT0`   | `OUT0_STATUS.RDY = 1` ∧ `OUT0_CTRL.IRQ_EN = 1`           |
| 3   | `OUT1`   | `OUT1_STATUS.RDY = 1` ∧ `OUT1_CTRL.IRQ_EN = 1`           |
| 4   | `TMR`    | `TMR_CTRL.PEND = 1` ∧ `TMR_CTRL.IRQ_EN = 1`              |

# Взаимодействие LOAD / STORE с MMIO

* `LOAD addr` / `STORE addr` при `addr ∈ [0xFFFF00, 0xFFFFFF]` обращаются к MMIO вместо обычной памяти.
* Чтение R-only-регистра возвращает текущее значение; запись игнорируется без эффекта.
* Чтение W-only-регистра возвращает 0; запись действует на устройство.
* Косвенные доступы через регистр `A` (`LDA+`, `LDA-`, `STA+`, `STA-`) ведут себя точно так же — диспетчеризация решается по итоговому адресу, а не по форме команды.
* `LOAD`/`STORE` к MMIO не имеют побочных эффектов, кроме описанных в семантике конкретного регистра. В частности, чтение `INx_DATA` снимает токен из FIFO — это единственный регистр с «деструктивным чтением».

(Потактовое описание этих циклов — в [SIGNALS.md](SIGNALS.md).)

# Прерывания и MMIO

Запрос принимается, если `PS.IE = 1` и `(IRQ_PENDING & IRQ_MASK) ≠ 0`. Среди установленных бит выбирается младший — это номер источника `k`.

Последовательность входа в обработчик (согласована с `RTI` из `ISA.md`):

```
1)  RSP ← RSP − 1;  M_R[RSP] ← FLAGS
2)  RSP ← RSP − 1;  M_R[RSP] ← PC
3)  PS.IE ← 0
4)  PC ← M[0x000000 + 4·k]      ; адрес обработчика из таблицы векторов
```

Обработчик обязан:

1. Прочитать данные из соответствующего регистра устройства (для входа — забрать токен через `LOAD INx_DATA`, для выхода — записать следующий байт в `OUTx_DATA`, для таймера — сбросить `PEND`).
2. Записать `1` в нужный бит `IRQ_ACK`, чтобы снять запрос.
3. Завершиться командой `RTI`, которая восстановит `PC` и `FLAGS` (включая `IE`).

# Пример: эхо stdin → stdout по прерыванию

Псевдоассемблер; адреса MMIO и таблицы векторов — символические константы.

```asm
            ; --- таблица векторов (в начале памяти данных) ---
            .org 0x000000
            .word in0_handler         ; вектор источника 0 (IN0)

            ; --- инициализация ---
            .org main
main:       PUSH 0x01
            STORE IN0_CTRL            ; IN0_CTRL.IRQ_EN ← 1
            PUSH 0x01
            STORE IRQ_MASK            ; разрешить только IN0
            EI
idle:       JMP idle                  ; ждать прерываний

            ; --- обработчик IN0 ---
in0_handler:
            LOAD  IN0_DATA            ; снять токен с входа
            STORE OUT0_DATA           ; отправить на выход
            PUSH 0x01
            STORE IRQ_ACK             ; сбросить pending бит 0
            RTI
```
