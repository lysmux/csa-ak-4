# Опкоды
Структура:
* Опкод — 1 байт
* Команды — 4 байта

## Системные
| Hex | Мнемоника | Псевдокод | Описание                                |
|-----|-----------|-----------|-----------------------------------------|
| 01  | NOP       | —         | Нет операции, увеличивается только `PC` |
| 02  | HALT      | stop      | Остановка процессора                    |

## Стек
| Hex | Мнемоника | Операнд | Стек ДО | Стек После | Псевдокод | Флаги | Описание                                   |
|-----|-----------|---------|---------|------------|-----------|-------|--------------------------------------------|
| 10  | PUSH      | imm     | :[]     | a:[]       | TOS ← imm | N Z   | Помещает на стек непосредственное значение |
| 11  | DUP       | —       | a:[]    | a:a:[]     | push TOS  | N Z C | Дублирует вершину стека                    |
| 12  | DROP      | —       | a:[]    | :[]        | pop       | N Z   | Снимает вершину стека                      |
| 13  | SWAP      | —       | a:b:[]  | b:a:[]     | TOS ↔ NOS | N Z   | Меняет местами два верхних элемента стека  |
| 14  | OVER      | —       | a:b:[]  | b:a:b:[]   | push NOS  | N Z   | Копирует второй элемент на вершину стека   |

## Прерывания
| Hex | Мнемоника | Операнд | Псевдокод                                                      | Описание                                                       |
|-----|-----------|---------|----------------------------------------------------------------|----------------------------------------------------------------|
| 70  | EI        | —       | PS.IE ← 1                                                      | Разрешить прерывания                                           |
| 71  | DI        | —       | PS.IE ← 0                                                      | Запретить прерывания                                           |
| 72  | RTI       | —       | PC ← M_R[RSP]; RSP ← RSP + 1; FLAGS ← M_R[RSP]; RSP ← RSP + 1; PS.IE ← 1 | Возврат из обработчика прерывания: восстановить `PC` и `FLAGS`, разрешить прерывания |

### Вход в обработчик прерывания

При установленном `PS.IE` и сигнале `IRQ` от устройства, после завершения текущей инструкции CU выполняет автоматически:

```
M_R[RSP] ← FLAGS; RSP ← RSP − 1
M_R[RSP] ← PC;    RSP ← RSP − 1
PS.IE ← 0
PS.IRQ ← 0
PC ← vector_table[N]
```

После чего исполнение продолжается с адреса обработчика. Возврат — через `RTI`.

# Регистры
## Системные
| Имя   | Назначение                                                           |
|-------|----------------------------------------------------------------------|
| PC    | Счётчик команд: адрес следующей инструкции в памяти команд           |
| PS    | Состояние процессора:                                                |
|       | • `EI` — разрешение прерываний                                       |
|       | • `IRQ` — флаг запроса прерывания                                    |
| IR    | Регистр инструкции: текущий опкод и его операнд после выборки        |
| AR    | Адресный регистр: адрес ячейки памяти данных, выставляемый на шину   |
| DR    | Регистр данных: буфер слова, читаемого/записываемого в память данных |
| TOS   | Регистр вершины стека данных                                         |
| NOS   | Регистр второго элемента стека данных                                |
| SP    | Указатель стека данных                                               |
| RSP   | Указатель стека возвратов                                            |
| FLAGS | Флаги результата и управления:                                       |
|       | • `N` — знак результата (старший бит)                                |
|       | • `Z` — нулевой результат                                            |
|       | • `V` — знаковое переполнение                                        |
|       | • `C` — перенос/заём (беззнаковое переполнение)                      |
